# mayan-kin v2 发布计划

`mayan-kin v2` 的目标不是继续堆概念，而是把项目稳定推进成一个
**可验证、可解释、可集成** 的玛雅天赋开源基础设施。

下面这份计划按“发布节奏”来写，不再只是开发想法列表。

## 发布原则

- 先补知识入口，再补解释层，再补产品化输出
- 先让普通用户看懂，再让专业用户可复用
- 先保证计算和样本稳定，再放大内容表达
- 每个版本都要有清晰的可见产出，而不是只写 TODO

## v0.4.0 知识入口补齐

### 目标

把神谕十字、地球家族、年度趋势、关系动力学四个入口写成真正可读的文档。

### 交付物

- `references/oracle.md`
- `references/earth-families.md`
- `references/yearly-fortune.md`
- `references/compatibility.md`

### 完成标准

- 每份文档都有概念说明
- 每份文档都有读法和适用场景
- 每份文档都能落到行动语言
- 不再出现只写“预留”“后续会补”的空文档

## v0.4.1 解释层统一

### 目标

把同一套知识入口统一成更适合对外输出的语言。

### 交付物

- 基础版解释
- 深度版解释
- 内容产品版解释
- 行动建议模板

### 完成标准

- 同一个盘可以生成不同风格的解释
- 术语能被翻译成现实建议
- 能回答“我接下来该怎么做”

## v0.4.2 年度与关系输出强化

### 目标

把年度趋势和关系动力学从基础解读升级成可追踪、可复盘的报告层。

### 交付物

- 年度趋势模板
- 关系动力学模板
- 个人说明书补强
- 合盘与年度联动说明

### 完成标准

- 年度内容不只是“今年的主题”，还能给出节奏建议
- 关系内容不只是“合不合”，还能给出互动和边界建议
- 可以直接用于咨询和内容输出

## v0.4.3 知识库结构化

### 目标

把文档从“可读”进一步推进到“可检索、可复用、可组合”。

### 交付物

- 统一的知识入口索引
- 结构化模板库
- 位置、调性、颜色、波符、家族的交叉说明

### 完成标准

- 任意一个解释入口都能被其他入口引用
- 文档之间不互相重复堆砌
- AI runtime 可以按主题拼装输出

## v0.5.0 对外可用版本

### 目标

让这个项目从“内部能用”变成“外部用户也能直接用”。

### 交付物

- 稳定的 CLI / JSON 输出
- 完整 README 首屏与 FAQ
- 示例集合
- 英文 landing 页

### 完成标准

- 普通用户能理解并上手
- 专业用户能拿去做咨询或内容产品
- 开发者能直接接入

## v1.0.0 开源基础设施版本

### 目标

把 `mayan-kin` 定位为一套长期可维护的玛雅天赋基础设施，而不是一次性的技能脚本。

### 交付物

- 稳定 API / CLI / JSON 契约
- 完整知识库与解释层
- 多运行时适配
- 可持续扩展的文档结构

### 完成标准

- 计算层稳定
- 文档层完整
- 解释层可复用
- 外部用户、咨询师、创作者和 AI agent 都能直接使用

## 推荐发布顺序

1. 先发布 `v0.4.0`，把四个知识入口补完整
2. 再发布 `v0.4.1`，统一解释语言和输出风格
3. 再发布 `v0.4.2`，强化年度和关系报告
4. 再发布 `v0.4.3`，把知识库结构化
5. 最后推进到 `v0.5.0` 和 `v1.0.0`
